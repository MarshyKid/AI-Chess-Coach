# ADR-0004 — Backend-First Development

## Status

Accepted

## Context

The eventual product will include a web frontend, conversational interface, board rendering, and user accounts.

However, the primary technical risk is not UI development.

The primary technical risk is generating meaningful and accurate chess coaching insights.

## Decision

Development will proceed backend-first.

Initial phases focus on:

- Replay engine
- Domain models
- FeatureStore
- Detectors
- Engine verification
- Pattern aggregation
- Weakness profiling

Frontend development should only begin after the chess intelligence engine has proven capable of producing useful coaching insights.

## Consequences

### Positive

- Focuses effort on core product value
- Faster validation of coaching concepts
- Frontend technology can change later without affecting analysis logic
- Easier testing

### Negative

- User interface arrives later
- Early demos may be CLI-based
