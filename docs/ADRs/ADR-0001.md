# ADR-0001 — LLM Does Not Perform Chess Analysis

## Status

Accepted

## Context

The system uses an LLM to provide personalized coaching and explanations.

There is a risk that the LLM may be used to analyze raw PGNs or determine chess correctness directly. This would introduce hallucinations, inconsistent results, non-deterministic behavior, and make testing difficult.

The goal of the project is to build a coaching system whose conclusions are grounded in structured chess analysis rather than language model reasoning.

## Decision

All chess correctness must originate from:

- Feature extraction
- Detectors
- Engine verification

The LLM may:

- Explain
- Summarize
- Teach
- Coach

The LLM may not:

- Analyze raw PGNs directly
- Determine whether a move is good or bad
- Determine whether a tactic exists
- Replace detectors
- Replace engine verification

The approved flow is:

PGN  
→ Feature Extraction  
→ Detectors  
→ Engine Verification  
→ Verified Events  
→ Pattern Aggregation  
→ Retrieval  
→ LLM Explanation

## Consequences

### Positive

- Deterministic analysis
- Easier testing
- Reduced hallucinations
- Consistent coaching output
- Easier debugging

### Negative

- Requires implementation of chess-specific detectors
- More engineering effort compared to an LLM-only solution
- New chess concepts require detector development
