# ADR-0002 — FeatureStore Is The Single Source Of Chess Facts

## Status

Accepted

## Context

Many detectors require access to the same board-level information:

- Piece safety
- Pinned pieces
- Mobility
- Pawn structure
- SEE values

Without a shared feature layer, detectors would duplicate calculations and potentially produce inconsistent results.

## Decision

FeatureStore is the canonical source of reusable chess facts.

All reusable board calculations must be implemented in FeatureStore.

Detectors should consume FeatureStore rather than calculating board facts independently.

Examples:

Good:

ForkDetector  
→ FeatureStore.piece_safety()

Bad:

ForkDetector recalculates attackers and defenders internally.

## Consequences

### Positive

- Reduced duplicated logic
- Consistent detector behavior
- Easier maintenance
- Easier optimization and caching
- Faster future detector development

### Negative

- Requires more upfront design
- FeatureStore may grow over time and require refactoring
