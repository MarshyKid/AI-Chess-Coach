# ADR-0005 — Event-Based Analysis Pipeline

## Status

Accepted

## Context

The system analyzes many games and produces long-term coaching insights.

A consistent intermediate representation is needed between game analysis and coaching.

## Decision

All player profiling must be built from aggregated events.

Pipeline:

Games  
→ DetectedEvents  
→ VerifiedEvents  
→ DetectedPatterns  
→ WeaknessProfile

The system should not create profiles directly from raw PGNs.

Events become the fundamental unit of analysis.

Examples:

- hanging_piece_created
- fork_missed
- knight_outpost_created
- king_safety_weakened

## Consequences

### Positive

- Reusable analysis artifacts
- Easier retrieval for coaching
- Easier analytics and reporting
- Enables future training recommendations
- Enables future machine learning systems

### Negative

- Requires careful event schema design
- Event storage may become large for users with many games
